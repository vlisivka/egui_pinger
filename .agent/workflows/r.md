---
description: Проведення технічних досліджень для проєкту (r)
---

Цей воркфлов використовується для проведення технічних досліджень, аналізу нових технологій та пошуку оптимальних рішень для проєкту.

### Основні принципи:
- **Мова**: Результати досліджень записуються **українською мовою** у файл `research.md`.
- **Обґрунтованість**: Кожне дослідження має базуватися на специфікації проєкту та поточному стані коду.
- **Досвід**: Пиши як професійний розробник продуктів із 10-річним досвідом.

### Кроки виконання:

1. **Аналіз вихідних даних**:
   - Уважно прочитати `Специфікація.md` для розуміння бізнес-вимог та технічних обмежень.
   - Ознайомитися з поточним кодом у `src/` для розуміння архітектури.
   - Перевірити `TODO.md` для виявлення завдань, що потребують попереднього дослідження.

2. **Проведення дослідження**:
   - Вивчити документацію сторонніх бібліотек або системних інструментів.
   - Провести експерименти (якщо потрібно) для перевірки гіпотез.
   - Сформулювати переваги та недоліки обраних підходів.

3. **Запис результатів у `research.md`**:
   - Якщо файлу не існує — створити його.
   - Структурувати звіт:
     - **Тема дослідження**: Назва проблеми.
     - **Мета**: Що саме ми хочемо з'ясувати.
     - **Опис варіантів**: Огляд можливих рішень. Старайся запропонувати два-три різних рішення.
     - **Рекомендації**: Найкращий варіант для реалізації з обґрунтуванням.
     - **Вплив на проєкт**: Які файли або модулі будуть зачеплені.

4. **Звіт**:
   - Коротко описати користувачу, що було досліджено та які висновки зроблено.
