description: Оновлення специфікації проєкту (s)
---

Цей воркфлов використовується для оновлення специфікації, документації та планів проєкту.

### Основні принципи:
- **Мова**: 
  - Інтерфейс та README — **англійська**.
  - Специфікація, Документація, TODO, Дослідження — **українська**.
- **Призначення проєкту**: Графічний інтерфейс для мережевих інструментів (ping, mtr, nslookup, traceroute) з акцентом на IP-телефонію (SIP).

### Кроки виконання:

1. **Аналіз вихідних даних**:
   - Ознайомитися з поточним станом коду в `src/`.
   - Прочитати існуючу документацію (`README.md`, `Специфікація.md`, `TODO.md`).

2. **Оновлення файлу `Специфікація.md`**:
   - Якщо файлу не існує — створити його.
   - Оновити розділи відповідно до нових вимог:
     - Підтримка декількох серверів одночасно (1.1.1.1, 8.8.8.8, SIP сервер).
     - Перевірка маршрутів через ICMP та UDP (зокрема порти SIP).
     - Перевірка DNS та аналіз затримок/тремтіння (jitter).
     - Режими роботи: "Швидкий" (активний пошук проблем) та "Повільний" (фоновий моніторинг).
     - Автоматична діагностика та генерація звіту про інцидент при виявленні аномалій.
     - Журнал подій та архів логів (з агрегацією/укрупненням старих даних).

3. **Синхронізація з `TODO.md`**:
   - Оновити список завдань у `TODO.md` згідно зі змінами в специфікації.
   - Використовувати українську мову.

4. **Оновлення `README.md`**:
   - Переконатися, що README актуальний та написаний **англійською мовою**.

5. **Звіт**:
   - Коротко описати внесені зміни користувачу.
