---
description: Створення детального плану дій для реалізації функцій (p)
---

Цей воркфлов використовується для розробки покрокових планів реалізації конкретних функцій або етапів проєкту.

### Основні принципи:
- **Мова**: Плани складаються **українською мовою**.
- **Розташування**: Файли планів зберігаються в каталозі `plans/`.
- **Формат**: Markdown з чіткою структурою та чек-лістами.
- **Досвід**: Пиши як професійний розробник програмного забезпечвення із 20-річним досвідом.

### Кроки виконання:

1. **Збір та аналіз інформації**:
   - Уважно прочитати `Специфікація.md` для розуміння кінцевої мети.
   - Прочитати `research.md` для врахування обраних технічних рішень.
   - Ознайомитися з `TODO.md` для визначення місця цього плану в загальній черзі завдань.
   - Уважно та детально проаналізувати поточний стан коду в `src/` для виявлення точок інтеграції та потенційних конфліктів.

2. **Створення плану в каталозі `plans/`**:
   - Назва файлу має бути описовою, наприклад: `plans/Plan - UDP Pinging.md`.
   - Структура плану повинна включати:
     - **Огляд**: Короткий зміст того, що буде зроблено.
     - **Зміни в архітектурі**: Які структури даних або модулі будуть додані/змінені.
     - **Етапи реалізації**: Детальний покроковий чек-ліст з конкретними діями.
     - **Критерії успіху**: Як перевірити, що план виконано успішно.

3. **Синхронізація**:
   - Переконатися, що кроки плану не суперечать `TODO.md`.

4. **Звіт**:
   - Презентувати створений план користувачу та описати ключові етапи.
