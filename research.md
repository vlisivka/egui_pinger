# Дослідження проблеми іконок у GNOME (Alma Linux 10)

## Тема дослідження: Відображення "шестерень" (gears) замість іконки в режимі Super (Activities)

## Мета
З'ясувати, чому при запуску `egui_pinger` на Alma Linux 10 (GNOME 46/47) у меню програм іконка відображається коректно, але в огляді запущених вікон (Super/Activities) відображається стандартна іконка шестерень.

## Опис варіантів

### Варіант 1: Відсутність StartupWMClass у .desktop файлі (Найімовірніша причина)
GNOME використовує властивість `StartupWMClass` у файлі `.desktop` для встановлення зв'язку між запущеним вікном (його `WM_CLASS` на X11 або `app_id` на Wayland) та відповідним лаунчером. 
Якщо цей зв'язок не встановлено:
- GNOME бачить вікно як "невідомий процес".
- Він не може зіставити вікно з файлом `.desktop`.
- Відповідно, він не знає, яку іконку взяти (параметр `Icon=` ігнорується для запущеного вікна).
- Результат: GNOME відображає іконку "за замовчуванням" (шестерні).

У нашому коді (`src/main.rs`):
```rust
eframe::run_native("egui_pinger", options, ...)
```
Це встановлює системний ID вікна як `egui_pinger`. Але у файлі `egui_pinger.desktop` відсутній запис `StartupWMClass=egui_pinger`.

### Варіант 2: Застарілий кеш іконок або десктоп-бази
Хоча іконка відображається у меню, GNOME Shell може мати проблеми з оновленням асоціацій для Wayland-вікон без явного індексування.
Установка в `~/.local` вимагає оновлення бази:
- `update-desktop-database ~/.local/share/applications`
- `update-icon-caches ~/.local/share/icons/hicolor`

### Варіант 3: Специфіка Wayland та GNOME 47
У нових версіях GNOME велике значення має `app-id`. Якщо воно не збігається з іменем файлу `.desktop` (без розширення), асоціація може зламатися.
Зараз:
- Файл: `egui_pinger.desktop`
- ID у коді: `egui_pinger`
Вони збігаються, але без `StartupWMClass` GNOME часто ігнорує цей збіг для двійкових файлів, запущених з `~/.local/bin`.

## Рекомендації
1. **Основне рішення**: Додати `StartupWMClass=egui_pinger` у файл `assets/linux/egui_pinger.desktop`. Це стандарт де-факто для вирішення проблеми "шестерень" у GNOME.
2. **Додатково**: Переконатися, що в скрипті інсталяції або в інструкції README додано команди оновлення системних баз іконок та десктоп-файлів для локального користувача.
