# План — Інструмент системної діагностики (System Tools)

## Огляд
Додати окреме вікно «Системні інструменти» (🔧), яке надає користувачу можливість запускати типові мережеві діагностичні команди прямо з графічного інтерфейсу **без потреби у терміналі**. Вікно містить покроковий довідник із діагностики мережевих проблем, комбобокс для вибору команди та поле виводу результатів.

Код реалізується як **окремий модуль** `src/ui/system_tools.rs`, щоб не ускладнювати вже великий `app.rs`.

## Зміни в архітектурі

### 1. Новий модуль `src/ui/system_tools.rs`
Містить:
- **Структуру `SystemCommand`** — опис однієї команди (назва, опис, програма, аргументи).
- **Функцію `get_commands() -> Vec<SystemCommand>`** — повертає список команд, відповідний для поточної ОС (`cfg!(target_os = "linux")` / `cfg!(target_os = "windows")`).
- **Структуру `SystemToolsState`** зі станом вікна:
  - `selected_command: usize` — індекс обраної команди.
  - `output: String` — результат виконання.
  - `is_running: bool` — прапорець «команда виконується».
  - `selected_tab: ToolsTab` — вкладка (Довідник / Команди).
- **Функцію `ui_system_tools_window()`** — малює вікно.
- **Функцію `run_command_async()`** — запускає команду через `std::process::Command` у фоновому потоці.

### 2. Новий модуль `src/ui/mod.rs`
Виділити UI-модулі в окремий каталог:
```
src/ui/
  mod.rs              // pub mod system_tools;
  system_tools.rs
```

### 3. Зміни в `src/app.rs`
- Додати поле `system_tools_open: bool` та `system_tools: SystemToolsState` до `EguiPinger`.
- Додати кнопку «🔧» у верхній панелі (біля перемикача тем).
- Виклик `ui_system_tools_window()` коли `system_tools_open == true`.

## Покроковий довідник (вкладка «Guide»)
Довідник структурується за сценаріями проблем. Кожен сценарій вказує, яку команду запустити та на що звернути увагу у виводі. Довідник перекладений через `tr!()`.

**Структура (скорочено):**

1. **Інтернет повністю не працює**
   - Перевірте: чи є IP-адреса? → `Мережеві інтерфейси`
   - Перевірте: чи є шлюз? → `Таблиця маршрутів`
   - Перевірте: чи працює DNS? → `DNS-запит google.com`

2. **Обриви та нестабільний зв'язок**
   - Подивіться сусідів та ARP → `ARP таблиця`
   - Перевірте статистику інтерфейсу → `Статистика мережевих інтерфейсів`

3. **Повільний інтернет**
   - Перевірте Wi-Fi: якість сигналу → `Стан Wi-Fi з'єднання`
   - Перевірте наявність встановлених з'єднань → `Активні TCP/UDP з'єднання`

4. **DNS не працює (сайти не відкриваються, але ping 8.8.8.8 працює)**
   - Перевірте настройки DNS → `Налаштування DNS`
   - Спробуйте запит вручну → `DNS-запит google.com`

5. **Потрібні права адміністратора** *(лише текст довідки, без кнопок)*
   - `sudo iptables -L -n` (Linux) — перевірка фаєрвола
   - `sudo tcpdump -i any -c 20` — перехоплення пакетів на рівні NIC

## Критерії успіху
- Кнопка 🔧 у верхній панелі відкриває вікно «System Tools».
- Вкладка «Guide» містить покроковий довідник із діагностики.
- Вкладка «Commands» дозволяє обрати команду, побачити її опис, запустити та побачити результат.
- Поле виводу починається з рядка `$ ip -c addr` (повна команда з аргументами).
- Інтерфейс не «замерзає» під час виконання команди.
- Всі тексти перекладені українською.
- Всі існуючі тести проходять.
